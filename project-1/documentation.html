Noteworthy:

  • Bulma Framework:
    • Very mobile friendly and very responsive. All elements scale properly to the screen size and a hamburger menu icon pops up to replace the navbar when the screen size gets small enough for mobile devices.
    • Utilizes different layout classes to help structure the page such as:
      1. Tiles which allows the user to map out their elements
      2. Section classes which are used to emphasize a section of elements by adding an indent and shifting everything to the left.
      3. Content classes which are used to structure the effected elements like a normal HTML page
  • Code:
    • All code is within 9 ES6 modules that follow DRY
    • Method for getting proper URL to send to the API has a parameter for general and specific searches
    • Each page's functionality has its own file for organizing methods and all are imported into init.js
  • User Experience:
    • Bulma loading class is utilized on the search button to let the user know a search is being performed
    • Favorite button disables once clicked and is kept track of to ensure users can't favorite the same card multiple times
    • Favorite button alerts user they can't perform the action when attempting to favorite the default card
    • Community page will show the user how many likes a character has and allows them to favorite any character in the list

Grading:

  • All base requirements were met
  • Went beyond requirements for firebase to include a likes counter for each character
  • App page includes a section to display most popular characters for users to check out
